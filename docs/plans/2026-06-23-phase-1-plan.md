# m1-eval Phase 1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the `m1-eval` core — a stepped, deterministic evaluator for M1 scripts — with the single-function and dependency-cone runners, a scenario input format, table lookup from `.m1cfg`, a library API, and a CLI, covering Tier-1 + Tier-2 builtins with Tier-3 IO stubbed.

**Architecture:** A tree-walking interpreter over `m1-core`'s CST, using `m1-typecheck`'s `Project`/`SymbolTable`/`resolve`/`ValueType` for the symbol model and name resolution. A per-tick value store plus per-call-site state runtime advance by an explicit `dt`; two runners (single-function, dependency-cone) choose which functions run each tick and produce a `Trace`. Builtins dispatch through one table that **fails loud** on anything unimplemented.

**Tech Stack:** Rust (edition 2024, rust-version 1.88), `clap` 4 (CLI), `serde` + `toml` + `serde_json` (scenario/trace I/O), `roxmltree` (`.m1cfg` value reader), `m1-core` and `m1-typecheck` as git-tag dependencies. Dev: `assert_cmd`, `tempfile`, `insta` (snapshots).

## Global Constraints

- **Dependency pins (git tags, per toolchain convention):** `m1-core` = `v0.12.0`, `m1-typecheck` = `v0.36.0`. Bump via consumer-bump PRs only; never depend on `@main`.
- **License:** `GPL-3.0-or-later`. Every source file carries the SPDX header `// SPDX-License-Identifier: GPL-3.0-or-later`.
- **Crate edition / MSRV:** `edition = "2024"`, `rust-version = "1.88"` (match `m1-doc`).
- **`publish = false`** (toolchain crates are git-tag, not crates.io).
- **Fail loud, never silently wrong:** any unimplemented builtin, unsupported construct, missing calibration cell, or unresolved symbol returns an `EvalError` — the evaluator never substitutes a guessed/default numeric value.
- **No proprietary text:** never copy text from the MoTeC M1 manuals into code, comments, or fixtures. Builtin semantics are paraphrased from understanding; cite behaviour in comments in our own words.
- **M1 identifiers may contain spaces** (e.g. `Filtering Cutoff Frequency`, `This.Debounce`) — never split identifiers on whitespace; only on `.` for path segments.
- **Determinism:** fixed tick grid + explicit `dt`; no wall-clock, no RNG. Same inputs ⇒ identical `Trace`.
- **No toolchain types leak past the public API:** `m1-core`/`m1-typecheck` types stay internal; the `lib.rs` surface exposes only `m1-eval`'s own types (mirror `m1-doc`'s boundary discipline).

**Discovery/loading pattern to mirror (from `m1-doc/src/loader.rs`):** recursively collect `*.m1scr` as `(basename, source)` pairs → `parse_all` → `Project::load` then `infer_return_types` → iterate `symbols()`.

**Resolved spec open-questions (decisions for this plan):**
1. **`.m1cfg` access:** a *dedicated* value reader is required (Task 3) — `with_config` keeps only shape/types, discarding axis breakpoints and body/parameter cell values.
2. **Dependency-cone graph:** `m1-eval` derives its own per-function read/write sets from the CST (Task 18) — `schedule.rs`'s sets are internal/unexposed.
3. **Scenario file schema:** TOML (primary) with an optional CSV time-series sidecar; JSON accepted via `serde` (Task 16).
4. **Call-site identity:** `(script_basename, byte_offset_of_call_node)` — stable across ticks for a fixed parse (Task 12).
5. **`motec-i2` vendoring:** not needed in Phase 1 (no `.ld`).
6. **`Trace` representation:** column-per-channel `Vec<Value>` aligned to a shared `Vec<f64>` time axis; `f32` storage acceptable later, `Value` for Phase 1 (Task 15).

// SPDX-License-Identifier: GPL-3.0-or-later
//! Fail-loud error type. The evaluator never substitutes a guessed value.

#[derive(Debug, Clone, PartialEq)]
pub enum EvalError {
    /// A builtin object/method we do not implement (Tier-3 or unknown).
    UnsupportedBuiltin { object: String, method: String },
    /// A syntactic construct the evaluator does not handle.
    UnsupportedConstruct { kind: String, at: usize },
    /// An identifier that resolves to no project symbol / local / builtin.
    UnresolvedSymbol { name: String },
    /// A calibration value (parameter or table cell) the .m1cfg did not provide.
    MissingCalibration { path: String },
    /// A type mismatch surfaced at runtime (e.g. arithmetic on a String).
    TypeError { detail: String },
    /// An input the scenario was required to provide but did not.
    MissingInput { channel: String },
    /// Wrong argument count/kind for a builtin call.
    BadCall { detail: String },
}

impl std::fmt::Display for EvalError {
    fn fmt(&self, f: &mut std::fmt::Formatter<'_>) -> std::fmt::Result {
        match self {
            EvalError::UnsupportedBuiltin { object, method } =>
                write!(f, "unsupported builtin: {object}.{method}"),
            EvalError::UnsupportedConstruct { kind, at } =>
                write!(f, "unsupported construct {kind} at byte {at}"),
            EvalError::UnresolvedSymbol { name } => write!(f, "unresolved symbol: {name}"),
            EvalError::MissingCalibration { path } => write!(f, "missing calibration value: {path}"),
            EvalError::TypeError { detail } => write!(f, "type error: {detail}"),
            EvalError::MissingInput { channel } => write!(f, "missing scenario input: {channel}"),
            EvalError::BadCall { detail } => write!(f, "bad call: {detail}"),
        }
    }
}
impl std::error::Error for EvalError {}
```

// SPDX-License-Identifier: GPL-3.0-or-later
use crate::error::EvalError;

#[derive(Debug, Clone, PartialEq)]
pub enum Value {
    Bool(bool),
    Int(i64),
    Uint(u64),
    Float(f64),
    Enum { id: usize, member: String },
    Str(String),
}

impl Value {
    pub fn as_f64(&self) -> Result<f64, EvalError> {
        match self {
            Value::Float(x) => Ok(*x),
            Value::Int(x) => Ok(*x as f64),
            Value::Uint(x) => Ok(*x as f64),
            other => Err(EvalError::TypeError { detail: format!("{other:?} is not numeric") }),
        }
    }
    pub fn as_bool(&self) -> Result<bool, EvalError> {
        match self {
            Value::Bool(b) => Ok(*b),
            other => Err(EvalError::TypeError { detail: format!("{other:?} is not boolean") }),
        }
    }
    pub fn truthy(&self) -> Result<bool, EvalError> { self.as_bool() }
}
```

## Testing Strategy

- **Unit tests per builtin** with hand-derived expected values (Tasks 13–19) — the Tier-2 time-domain math is the highest-risk surface; every operator family has an explicit-number test.
- **Deterministic snapshot tests** (`insta`) on `Trace` output for the `tests/fixtures/mini` project (Task 25/27) — same inputs ⇒ identical snapshot.
- **Fixture realism check:** Task 3/14 carry a NOTE to validate the `.m1cfg` parser and table layout against a trimmed snippet of the real `~/Documents/UQR/code/EV-M1/parameters.m1cfg` before finalising — but committed fixtures stay synthetic (no proprietary/calibration data in the repo).
- **CLI integration tests** (`assert_cmd`, Task 29) — exit codes, output files, `--coverage`.
- **Fail-loud tests:** every error variant in `EvalError` has at least one test that provokes it (unsupported builtin, missing calibration, unresolved symbol, missing input, arity mismatch).
- **Cross-check (manual, fidelity):** once Phase 1 runs a real EV-M1 control script, spot-check a few channel values against MoTeC M1 Sim (validation-oracle role) and record divergences; full golden tests are a fidelity follow-up, not a Phase-1 gate.

## Out of scope for Phase 1 (do NOT build here)

Whole-project multi-rate scheduler; user-function call evaluation across functions (beyond the cone's ordered scheduled functions); MoTeC log import (CSV/`.ld`); counterfactual override + diff; LSP hover/inlay; `motec-i2` integration.
